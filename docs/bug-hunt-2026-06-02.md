## REAL BUGS

### B1 — Dark-mode soft-tint contrast collapse (HIGH severity)

**Symptom.** In dark mode, `.pc-av` avatar text unreadable (1.04:1 contrast).
Breaks 5 tinted backgrounds.

**Root cause.** `theme.js:applyTeamTheme()` overrides `--team-primary-soft` /
`--team-secondary-soft` with `mixHex(color, '#ffffff', 0.90)` regardless of
theme. Inline `style.setProperty` beats `[data-theme="dark"]` block in
`tokens.css`.

**Blast radius.** `.pc-av`, `.wc-row.me`, `.gb-tag` + `.st-row.me .st-bar i`,
keystone heading hint, `.idx-item.active`.

**Fix.** Branch on `isDark` in `applyTeamTheme`. One-liner per token.

---

### B2 — Direct-load deep-link to `#stat-<slug>` lands on Overview (MED)

**Symptom.** Fresh load of `index-v2.html#stat-xwoba` lands on Overview, not
Stat School. Round 1 passed because it tested navigation via hashchange;
Round 2 caught the direct-load case.

**Root cause.** `parseTabFromHash` in `render.js` falls back to `overview`
for non-tab hashes. No hashchange fires on initial load, so per-tab module
listeners don't get a chance to override.

**Fix.** `parseTabFromHash` should recognize `stat-` and `player-` prefixes.

---

### B3 — Duplicate element ID `stat-mlb_rank` (MED)

**Symptom.** Two DOM elements share the ID — Stat School keystone and a
per-stat card. Invalidates HTML; `getElementById` returns only first.

**Fix.** Drop `mlb_rank` entry from `stat_school.json.stats` (keystone
already covers the concept).

---

### B4 — Browser back doesn't close modal (LOW)

**Symptom.** Modal open → press back → hash reverts but modal stays.

**Fix.** `players.js` hashchange handler should `closeModal()` when hash
leaves `#player-`.

---

### B5 — No `<h1>` on any tab (MED — a11y)

**Symptom.** Audit shows zero `<h1>` elements rendered across all 4 tabs.
Brand title in header is a `<b>`. Screen-reader users hit "no main
heading on page."

**Fix.** Change `<b id="brand-title">` to `<h1 id="brand-title">` in
`index-v2.html`. CSS already sets brand size/weight — visual unchanged.
Add a screen-reader-only `<h1>` per tab if a per-tab top heading is
preferred, OR leave brand as the page h1.

---

### B6 — Overview tab has no `<h2>` (MED — a11y + consistency)

**Symptom.** Heading audit:
- Overview: h3, h3, h3, h3, h3, h3, h4, h4 (no h2)
- Players: h2 "Players" + h3s for role groups ✓
- Team Stats: h2 "Team Stats" + h3 ✓
- Stat School: h2 "Stat School" + h3 ✓

Overview's "The season right now" eyebrow is a `<p>`, not a heading.

**Fix.** Make the Overview top heading an `<h2>` (visually a hidden "Overview"
heading via `.sr-only`, or replace the eyebrow with a styled `<h2>`).

---

### B7 — Horizontal scroll at 320px viewport (LOW)

**Symptom.** At 320px (iPhone SE), `document.scrollWidth=385px` vs
`window.innerWidth=320px`. Content forces horizontal scroll.

**Note.** Design only spec'd down to 480px. Outside the stated breakpoint.
Could be from a min-width on some element (e.g., `.kpi`) or text reflowing
poorly.

**Fix priority.** Low — out of design scope, but easy fix likely (add
`min-width: 0` or `flex-wrap` somewhere strategic).

---

### B8 — Stat School blank during `stat_school.json` load (LOW — UX)

**Symptom.** With slow network, clicking the Stat School tab while
`stat_school.json` is still loading shows an empty body — no skeleton.

**Root cause.** `stat-school.js:render()` returns a promise; no placeholder
content is rendered before the JSON resolves.

**Fix.** Inject a skeleton block synchronously at render() start, replace
on JSON load.

### M4 — Recent-games away game `"@ at OPP"` double prefix

`assets/overview.js:388,395`. Away game meta has `prefix='@ '` and concatenates
with literal `'at '`. Fix: drop the prefix variable.

### M5 — Dead code: `JaysFormat.timeOfDay`

Defined in `assets/format.js`, never referenced externally.

### M6 — `ordinal(-1)` returns `"-1th"` (round 3 finding)

Nonsense for ranks (ranks are 1–30). Probably should return `"—"` for
non-positive integers.

### M7 — `relativeAge(future)` returns `"just now"` (round 3 finding)

Future dates produce negative diffs which fall under the `< 60s` branch.
Future timestamps shouldn't happen for RSS items but defensive handling is
cheap.

### M8 — `ordinal("abc")` returns `"—"`, Python version returns input (round 3)

JS `ordinal` short-circuits to `'—'` for unparseable. Python sibling returns
`str(n)` (i.e. the input string). Inconsistency; no real-world impact since
ranks are always numeric.

### M9 — No `@media print` stylesheet (round 3)

Print rendering shows the live UI verbatim — header / scrim / sticky bars
all render. Out of scope per current plan; document as v2.x.

## What round 3 specifically taught us (survivorship-bias retrospective)

Round 1 + Round 2 collectively tested **navigation**, **interaction**,
**data shape**, **responsive**. They found 4 real bugs.

Round 3 explicitly hunted in **untouched** surface:
- **Formatter edge inputs** → found 3 polish items (ordinal/relativeAge edge cases)
- **Unusual data shapes** → all passed (no is_us, no rotation, no pos, in-progress
  games, malformed news) — the renderers are defensively resilient
- **Accessibility deep-dive** → found 2 real a11y bugs (no h1, no h2 on Overview)
- **Loading-state visibility** → found 1 UX bug (Stat School blank during load)
- **Very narrow viewport** → found 1 below-spec bug (320px overflow)
- **Print / reduced-motion / globals / SVG namespace / pill contrast / heap leak**
  → all passed

The pattern: **the things round 3 found were ALL in surfaces that visual review
naturally skips** (heading semantics, fractional-second animations, accent-stripped
slugs, edge-cased input handlers, async-load empty states). The runtime behavior
under happy-path data is solid — the gaps are at the margins.

If round 4 happens, target: i18n, RTL languages, very long text overflow,
locale-specific number/date formatting, in-place mutation of `state` between
tab renders, scroll-restoration on tab switch (does the page remember where you
were?), and keyboard navigation through tab content (not just chrome).
